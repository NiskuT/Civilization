164  /* Representation of the Thumb IT state. ITSTATE is a 32-bit
165  value with 4 8-bit lanes. [7:0] pertain to the next insn to
166  execute, [15:8] for the one after that, etc. The per-insn
167  update to ITSTATE is to unsignedly shift it right 8 bits,
168  hence introducing a zero byte for the furthest ahead
169  instruction. As per the next para, a zero byte denotes the
170  condition ALWAYS.
171 
172  Each byte lane has one of the two following formats:
173 
174  cccc 0001 for an insn which is part of an IT block. cccc is
175  the guarding condition (standard ARM condition
176  code) XORd with 0xE, so as to cause 'cccc == 0'
177  to encode the condition ALWAYS.
178 
179  0000 0000 for an insn which is not part of an IT block.
180 
181  If the bottom 4 bits are zero then the top 4 must be too.
182 
183  Given the byte lane for an instruction, the guarding
184  condition for the instruction is (((lane >> 4) & 0xF) ^ 0xE).
185  This is not as stupid as it sounds, because the front end
186  elides the shift. And the am-I-in-an-IT-block check is
187  (lane != 0).
188 
189  In the case where (by whatever means) we know at JIT time
190  that an instruction is not in an IT block, we can prefix its
191  IR with assignments ITSTATE = 0 and hence have iropt fold out
192  the testing code.
193 
194  The condition "is outside or last in IT block" corresponds
195  to the top 24 bits of ITSTATE being zero.
196  */
197  UInt guest_ITSTATE;
